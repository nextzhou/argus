# Argus V2 产品概述

AI Agent（如 Claude Code、Codex、OpenCode）已经成为软件开发中的重要工具。然而，随着项目复杂度的增加，如何让 AI Agent 在复杂工程任务中保持可靠、可预期的行为，成为团队面临的实际挑战。Argus V2 是一个专为 AI Agent 设计的工作流编排工具，通过结构化的流程管理和持续的状态监控，让 AI Agent 在复杂软件工程中更稳健、更可控。

## 为何 AI Agent 需要编排？

在实际的软件开发场景中，AI Agent 在处理复杂工程任务时面临以下挑战：

1. **缺乏结构化的引导**：复杂的任务（如一个完整的版本发布流程）通常包含环境自检、代码静态分析、单元测试、安全审计、文档更新以及最后的发布操作。AI Agent 如果缺乏明确的步骤指引，可能会遗漏关键环节，或者在错误的顺序下执行操作。
2. **缺乏持久的状态感知**：目前的 AI Agent 多以会话为中心，虽然有一定的上下文记忆，但对于跨会话、长周期的项目进度缺乏天然的跟踪机制。这导致在多人协作或多次迭代中，Agent 很难准确识别当前任务的所处阶段。
3. **缺乏项目不变量的守护**：项目标准和规范（如必须通过的代码风格检查、必须同步的接口文档）往往散落在各种文档中。AI Agent 在进行修改时，很难自发地、持续地维护这些不变量。当规范被无意破坏时，往往只有在最后的集成阶段才能被发现。

Argus V2 通过在 AI Agent 之上构建一层编排层，有效地解决了这些问题。

## Argus V2 的核心定位

Argus V2 并非 AI Agent 的替代品，而是它们的协调层。它的定位非常明确：提供结构化的流程框架，而不介入具体的业务执行。

Argus V2 不直接执行具体的业务逻辑，也不代替 AI Agent 进行决策。它的核心职责是：告诉 AI Agent 应该做什么，提供必要的背景知识和任务指引，然后由 AI Agent 利用其推理和执行能力去完成任务。

Argus V2 的设计遵循"产物即真相"的原则。它不依赖于复杂的内部数据库或中间状态，而是通过直接检查项目中的实际产物（如文件、目录、提交记录等）来判断项目的真实状态。这种设计确保了 Argus V2 能够与现有的开发工具链无缝集成，并且具有极高的容错性和透明度。

## 三大核心概念

为了实现高效的编排，Argus V2 引入了三个核心概念：工作流、不变量与流水线。

### 工作流（Workflow）

工作流是 Argus V2 中用于定义任务执行路径的组件。它采用过程式的思维，将一个复杂的、多步骤的任务拆解为一系列有序的任务单元。每个任务单元都明确了 AI Agent 需要执行的具体动作、需要使用的技能以及需要达到的目标。

通过工作流，管理者可以将团队的最佳实践沉淀为标准的可执行规程。AI Agent 在执行工作流时，会按照预定的顺序逐个完成任务。每完成一个步骤，Argus V2 都会记录进度并自动下发下一个任务的指令。这种方式不仅降低了 Agent 的出错率，也极大地提高了任务的可观测性。

### 不变量（Invariant）

不变量是 Argus V2 的声明式状态守护机制。与工作流关注"如何做"不同，不变量关注的是"结果应该是怎样"。它是一组条件，描述了项目在任何时候都应该满足的状态。

例如，"项目必须完成初始化"、"核心代码必须通过静态分析"、"接口文档必须是最近更新的"等，都是典型的不变量。Argus V2 会持续监控这些不变量，一旦发现项目状态偏离了预期（例如开发者手动修改了配置导致规范失效），它会立即识别出这种偏离，并引导 AI Agent 执行相应的修复流程。

### 流水线（Pipeline）

流水线是工作流的具体执行实例。当一个工作流被启动时，Argus V2 就会创建一个与之对应的流水线记录。

流水线负责实时跟踪当前任务的进度，记录每个步骤的开始时间、结束时间以及产生的输出信息。它为跨会话的任务协作提供了数据支撑。无论是在会话中断后重新启动，还是在不同的开发环境中接续任务，流水线都能确保 AI Agent 能够从正确的位置继续工作，不会出现进度丢失或重复执行的情况。

## 关键能力

Argus V2 为 AI Agent 的工程化应用提供了以下核心能力：

1. **工作流编排与自动推进**：Argus V2 能够将复杂的工程目标拆解为 AI Agent 可理解、可执行的细粒度任务。它支持任务间的有序流转，实现任务处理的闭环。
2. **项目状态声明式守护**：通过不变量系统，Argus V2 实现了对项目健康度的持续监控。它在不干扰开发者正常工作的前提下，自动识别项目中的潜在风险并引导修复。
3. **跨 Agent 统一集成**：目前支持 Claude Code、Codex 以及 OpenCode 等主流工具。Argus V2 确保了无论用户使用哪种 Agent，都能获得一致的工作流体验和状态约束。
4. **项目健康诊断**：Argus V2 提供了系统化的健康诊断能力。它能够扫描项目的配置环境、工作流定义、日志记录等多个维度，快速定位集成问题或配置错误。诊断工具遵循"只诊断、不干预"的原则，清晰展示问题根源并给出修复建议，最终决策权始终保留在开发者手中。
5. **多项目工作区管理**：针对个人开发者和团队，Argus V2 支持多项目的工作区管理。用户可以将多个关联的项目注册到一个统一的工作区中，实现跨项目的统一引导。
6. **团队协作支持**：Argus V2 的所有核心配置都采用了易于版本控制的格式，可以通过 Git 在团队成员之间共享。团队成员可以共享同一套编排逻辑和质量标准，项目配置一次完成后，所有成员均可受益。

## 运作方式

Argus V2 通过与 AI Agent 的集成机制实现编排。每当用户向 AI Agent 发送消息时，Argus V2 会在后台进行一次状态检查：它扫描当前是否有进行中的工作流、检查项目的不变量是否满足，然后将必要的上下文信息（当前任务指令、进度提醒、异常警告等）注入到 Agent 的对话中。Agent 接收到这些上下文后，按照指引执行任务，完成后向 Argus V2 报告结果，Argus V2 随即记录进度并自动下发下一个任务。这一过程在用户几乎无感知的情况下持续运转，确保复杂任务始终按照预定轨道推进。

## 典型使用场景

Argus V2 可以广泛应用于各种软件开发场景，以下是几个典型的例子。

**项目环境就绪度保障**：在新项目初始化或新成员加入项目时，确保每个人的开发环境、依赖库、预设规则都完全一致。Argus V2 可以自动检测环境缺失项，并引导 AI Agent 一步步完成安装和配置，避免"在我机器上能跑"的经典问题。

**代码质量与一致性维护**：通过定期检查项目是否近期进行过代码审计或单元测试，Argus V2 可以在相关检查过期时提醒 AI Agent 在下一次提交前先执行必要的质量验证，将质量保障融入日常开发流程。

**文档与规范的闭环管理**：Argus V2 可以自动识别出可能由于代码修改而需要同步更新的文档。当检测到文档可能已过期时，它会引导 AI Agent 主动发起文档审查流程，防止文档与代码长期脱节。

## 支持的 AI Agent

Argus V2 目前原生支持以下三款主流 AI Agent：

- **Claude Code**（Anthropic）
- **Codex**（OpenAI）
- **OpenCode**

三款 Agent 均通过各自的集成机制与 Argus V2 协同工作。无论用户使用哪款 Agent，看到的工作流进度和受到的状态约束都完全一致，切换 Agent 不会影响任何进行中的任务。
